# OpenCode 오케스트레이터 플러그인 (KO)

> **[OpenCode](https://opencode.ai)를 위한 멀티 에이전트 협업 시스템**

## 🏛️ 프로젝트 철학: 끝장 실행 전략 (Relentless Execution)

우리는 "빠른" AI보다 **"정확한"** AI를 믿습니다. 우리의 에이전트들은 끈질깁니다. 에러가 발생해도 멈추지 않고, 전략을 수정(Pivot)하고 다시 계획을 세워 목표가 완벽하게 달성될 때까지 실행합니다.

### 5단계 핵심 아키텍처

1.  **🧠 선 분석 (Think First)**: 무작정 코드를 짜지 않습니다. 에이전트는 먼저 모든 문서를 읽고 프로젝트의 핵심과 경계를 요약하여 보고합니다.
2.  **🌲 계층적 DAG (Hierarchical Planning)**: 전체 업무를 큰 그림(Architecture)에서 세부 마이크로 태스크로 계층적으로 분해하여 지도를 그립니다.
3.  **👥 병렬 실행 (Parallel Execution)**: 설계된 지도를 바탕으로 의존성이 없는 작업들을 여러 에이전트가 동시에 수행하여 효율을 극대화합니다.
4.  **🛡️ 글로벌 싱크 게이트 (Global Sync Gate)**: 병렬 작업이 끝나면 'Style Guardian'이 투입되어 전체 파일 간의 정합성과 로직 일관성을 최종 검증합니다.
5.  **⏳ 끝장 실행 (Relentless Completion)**: 시간 제한을 두지 않습니다. 100% 검증된 결과(PASS)가 나올 때까지 자가 치유 루프를 돌며 완벽을 추구합니다.

## 🚀 주요 특징
- **🧩 병렬 DAG 오케스트레이션** - 독립적 작업의 동시 수행
- **🎯 고정 모델 최적화** - 저성능 모델로도 시니어급 결과물 보장
- **🛡️ 스타일 가디언** - 프로젝트 전체의 코드 일관성 강제
- **🦀 Rust 기반 코어** - 고성능 네이티브 검색 및 분석

## 🛠️ 사용 방법
```bash
/dag "JWT 기반 사용자 인증 기능 구현해줘"
```
에이전트 군단이 분석부터 설계, 병렬 구현, 최종 싱크 검증까지 시시포스처럼 완벽하게 수행합니다.

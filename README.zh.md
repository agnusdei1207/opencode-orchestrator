# OpenCode 编排器插件 (ZH)

> **[OpenCode](https://opencode.ai) 的多智能体协同系统**

## 🏛️ 项目哲学：西西弗斯策略 (The Sisyphus Strategy)

我们不追求“快速”的 AI，我们追求 **“正确”** 的 AI。就像西西弗斯一样，我们的智能体永不言弃。它们在遇到错误时不会停止，而是调整策略（Pivot），重新规划，直到任务达到 100% 完美的成功。

### 五大核心架构阶段

1.  **🧠 深度分析 (先思考)**：不盲目开始编写代码。智能体必须先阅读所有文档，总结项目的核心逻辑和边界。
2.  **🌲 层级 DAG 规划 (层级规划)**：将任务从宏观架构分解为微观任务，生成有向无环图 (DAG)。
3.  **👥 并行执行 (并行流)**：基于任务图，多个智能体同时处理互不干扰的任务，极大提升效率。
4.  **🛡️ 全局同步门控 (全局同步)**：并行任务完成后，由 'Style Guardian' 进行全局一致性检查，确保导入、导出和逻辑模式完全对齐。
5.  **⏳ 终极交付 (永不言弃)**：不设人为的时间限制。目标只有唯一一个：100% 通过验证 (PASS) 的完美代码。

## 🚀 主要特性
- **🧩 并行 DAG 编排** - 独立任务并发执行
- **🎯 固定模型优化** - 即使使用低性能模型也能保证专家级输出
- **🛡️ 风格守护者** - 强制执行全项目的代码一致性
- **🦀 Rust 驱动核心** - 高性能原生搜索与分析

## 🛠️ 如何使用
```bash
/dag "实现基于 JWT 的用户认证功能"
```
智能体军团将像西西弗斯一样，从分析到设计、并行实现以及最终同步验证，完美交付任务。
